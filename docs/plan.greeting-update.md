### 매핑 의도

- **body 5개 단락** (질문 → 답 → H 다섯 약속 → 배려 수단 → 송도 다짐) — Greeting 본문 영역에 자연스럽게 흐름.
- **closing 1줄** (마무리 인사) — 디자인상 closing은 별도 섹션에서 가운데 정렬 + 골드 바 + 큰 폰트로 강조됨. 짧고 굵은 인사가 그 디자인에 가장 잘 맞음.
- 기존 `closing`("단순히 질병을 고치는 곳이 아닌, 환자분이 건강한 일상을 행복하게 영위하실 수 있는…") 은 새 docx에 없으므로 교체.
- 기존 `body[0..1]`("저희는 치아 하나를 치료하는 의술을 넘어…", "구강악안면외과와 치과보존과 전문의 협진을 통해…") 도 새 docx로 완전 교체.

## 4. 추가 검토 — `app/about/page.tsx` 의 `metadata.description`

현재:
```
description: '진료 너머, 사람의 고귀함을 생각합니다. 구강악안면외과 · 치과보존과 전문의 협진으로 정교한 진료에 환자를 향한 깊은 시선을 더합니다.',
```

새 인사말의 핵심 어휘 ('Heart·Human·Healing·Heritage·High-end', '의식하 진정요법(수면치료)', '인천 송도') 가 들어 있지 않음. 그러나 description 은 SEO·소셜 카드에 노출되는 영역이고, 메타 변경은 "의원소개 인사말 수정" 의뢰 범위 밖.

## 6. 트레이드오프 / 리스크

- **리스크 1**: body 단락이 길어지면서 about 페이지의 좌(이미지)·우(글) 그리드에서 우측이 이미지보다 훨씬 길어짐. 다만 mobile에서는 1열로 떨어지고, desktop에서도 `align-items: center` 라 자연스러움. 시각적으로 글이 더 깊게 흐르는 디자인이 됨 (오히려 인사말 페이지로서는 적절).
- **리스크 2**: docx 본문에 큰따옴표(`""`) 가 사용됨 — 코드 문자열에 그대로 들어가도 ES 문자열 리터럴(작은따옴표) 안이라 충돌 없음. 한국어 큰따옴표는 그대로 유지(시각적으로도 자연스러움).
- **리스크 3**: 기존 closing 의 톤("당신이 머무는 이 순간이 가장 편안하도록, 진심을 다하겠습니다.")이 새 closing 보다 호소력 있음. 새 closing 은 짧고 평이한 마무리 인사. → docx 가 사용자 의뢰의 정본이므로 그대로 반영.
